4-1. ZeroLlama 🔒 — 소비자용 GPU의 로컬 모델 시나리오를 위해 만든 에이전틱 코딩 클라이언트 (VS Code 확장)

  • 무엇: 소비자용 GPU(4~24GB)에서 양자화된 로컬 모델로 코딩하기 위한 에이전틱 코딩 클라이언트. 기존 에이전트/런타임은 클라우드 대형 모델을 전제로 설계돼 이 환경에서 제약이 큰데, 그 격차를 아래 레이어로 메웠다.
  • 기술: TypeScript · llama.cpp · 🔒 비공개 (pre-alpha) · src/zerollama/ 아래 35개 모듈 27,515 LOC(테스트 제외) 서브시스템
  • 차별점 — 소비자용 GPU·양자화 모델 시나리오 전용 레이어:
    • 자동 튜닝 — 복잡한 옵션을 몰라도 GPU 사양·VRAM에 맞춰 적합한 모델·양자화를 제안하고, KV 캐시 크기·GPU에 올릴 레이어 수(ngl)·컨텍스트 크기·flash-attn 등을 최적으로 설정한다. 왜 그렇게 잡았는지 근거도 함께 보여주고(원하면 직접 조정 가능), MoE 모델이 OOM 나면 전문가(expert) 레이어를 CPU RAM으로 오프로드해 8GB VRAM으로도 35B급(활성 ~3B) 모델을 구동한다 — 활성·공유 부분만 GPU에, 전문가는 CPU RAM에.
    • 양자화 모델용 도구 재설계 — 작은 모델·작은 컨텍스트(8~32K)에서도 실제로 쓸 수 있도록 도구를 다시 만들었다. 시스템 프롬프트를 통상의 1/8로 줄여 사고할 공간을 확보하고, 편집 블록을 9가지 방식으로 대조해 공백·들여쓰기가 조금 어긋나도 적용되게 하고, 추론(<think>) 블록은 다음 턴 입력 전에 제거하며, 코드 수정 후 ESLint·TypeScript·ruff·cargo·go를 돌려 에러를 모델에 되돌려준다.
    • 도구 호출에 의존하지 않는 메모리 — 작은 모델이 메모리 도구 호출을 자주 빠뜨리는 문제를 피하려 저장·회수 판단을 모델 밖에서 처리한다. 자동으로 저장하고 매 턴 관련 기억을 프롬프트에 붙여주되, 노이즈를 걸러 불필요한 회수를 크게 줄이면서(측정상 회수 토큰 −92%) 정작 필요한 기억은 유지한다. 임베딩 모델 다운로드 없이 SQLite FTS5로 구현.
    • 작은 컨텍스트에 모든 입력을 담는 설계 — 8~32K 창에 프롬프트·기억·코드·로그가 다 들어가도록, 로그는 핵심 줄만 요약하고(5,000줄→100줄 수준), 저장소는 tree-sitter+PageRank로 중요도를 매겨 예산 안에 포함하고, 창이 차오르면 마무리를 유도하거나 대화를 압축한다. 입력이 차지하던 공간을 줄여 추론에 쓸 여유를 확보한다.
    • 폰 원격 (계정 불필요) — PC와 폰을 6자리 PIN·QR로 한 번 연결하면 폰 브라우저에서 진행 상황 확인·도구 실행 승인·모델 전환. 별도 클라우드 계정을 만들지 않는다.
  • 엔지니어링: 하드웨어 감지 NVIDIA(nvidia-smi)/AMD·Vulkan/Apple Metal, JIT 프로비저너(모델 선택→llama.cpp 다운로드→인자 튜닝→채팅 배선 자동화), 실제 import를 grep해 미연결 모듈을 표시하는 자가 감사 리포트. 대부분 모듈이 자체 단위 테스트 포함.
